## 一、项目概述

### 1.1 项目名称
市场咨询任务跟踪工具

### 1.2 项目目标
开发一款面向市场咨询场景的任务跟踪管理工具，实现任务信息解析、记录、查询、跟踪、提醒的一体化管理，提升市场咨询工作的效率和质量。

### 1.3 运行环境
- **操作系统**：Windows
- **技术栈**：Python + PyQt5
- **数据库**：SQLite（本地存储）
- **打包形式**：可执行程序（.exe）

### 1.4 技术集成
| 集成项 | 实现方式 |
|--------|----------|
| OCR识别 | 本地Tesseract OCR（优先）+ 预留云端API接口 |
| Outlook邮件 | 直接连接Outlook读取 + .msg文件导入 |
| MSG邮件文件 | extract-msg库解析 + 无第三方库备用方案 |
| 企业微信 | 自建应用发消息 + webhook群机器人 |
| 数据导出 | Excel表格 |
| **学习积累** | **基于任务数据的智能学习与知识积累** |

## 二、功能需求

### 2.1 内容导入功能

#### 2.1.1 功能说明
支持多种渠道的内容导入，自动解析为任务信息。

#### 2.1.2 导入方式
| 方式 | 说明 | 优先级 |
|------|------|--------|
| 文本粘贴 | 直接粘贴文字内容 | P0 |
| 图片导入 | OCR识别（本地Tesseract优先） | P0 |
| Outlook邮件 | 直接连接Outlook指定邮件读取 | P0 |
| MSG邮件文件 | 读取导出的.msg文件，解析邮件内容 | P0 |
| 企业微信 | 聊天记录文件导入 + 截图OCR识别 | P1 |

#### 2.1.3 MSG邮件文件导入功能

##### 2.1.3.1 功能概述
支持导入Outlook导出的MSG格式邮件文件，自动解析邮件内容并提取任务相关信息。

##### 2.1.3.2 技术实现
| 组件 | 实现方式 | 说明 |
|------|----------|------|
| 首选方案 | extract-msg库 | 功能完整，解析准确 |
| 备用方案 | 自定义解析器 | 无第三方库时使用 |
| 文件大小限制 | 100MB | 防止大文件导致内存溢出 |

##### 2.1.3.3 MSG邮件解析字段
| 字段 | 说明 | 来源 |
|------|------|------|
| 任务名称 | 邮件主题或前50字符 | subject |
| 任务具体内容 | 邮件正文 | body |
| 咨询者姓名 | 发件人姓名 | sender_name |
| 咨询者联系方式 | 发件人邮箱 | sender_email |
| 创建时间 | 邮件发送时间 | sent_date |

## 四、学习积累功能需求（V2.0新增）

### 4.1 功能概述

学习积累功能从所有任务数据中自动学习和提取有价值的信息，丰富通讯录和推荐库。

**核心目标：**
- 从历史任务中自动学习咨询者和答复人信息
- 从历史任务中提取答复人与关键模块的关联关系
- 为智能推荐提供数据支撑

### 4.2 通讯录学习

#### 4.2.1 学习内容

| 学习类型 | 学习字段 | 数据来源 | 说明 |
|----------|----------|----------|------|
| 咨询者信息 | 姓名、部门、公司 | Task表 | 从任务的咨询者字段学习 |
| 咨询者联系方式 | 手机号、邮箱 | Task表 | 从任务的咨询者联系方式学习 |
| 答复人信息 | 姓名、部门 | Task表 | 从任务的答复人字段学习 |
| 任务频次统计 | 任务次数 | Task表 | 统计每人被咨询的次数 |
| 行业分布 | 所属行业 | Task表 | 统计咨询者所在行业 |

#### 4.2.2 学习记录表结构

**表1：contact_learning_records（单次学习记录）**
```sql
CREATE TABLE contact_learning_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    contact_name VARCHAR(100) NOT NULL,
    contact_type VARCHAR(20) NOT NULL,  -- '咨询者'/'答复人'
    department VARCHAR(100),
    company VARCHAR(200),
    phone VARCHAR(50),
    email VARCHAR(100),
    industry VARCHAR(100),
    source_task_id VARCHAR(50),
    learned_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(contact_name, contact_type, source_task_id)
);
```

**表2：learned_contacts（综合联系人表）**
```sql
CREATE TABLE learned_contacts (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    contact_name VARCHAR(100) NOT NULL UNIQUE,
    contact_type VARCHAR(20) NOT NULL,  -- '咨询者'/'答复人'
    department VARCHAR(100),
    company VARCHAR(200),
    phone VARCHAR(50),
    email VARCHAR(100),
    industry VARCHAR(100),
    task_count INTEGER DEFAULT 1,  -- 任务次数
    confidence FLOAT DEFAULT 0.5,  -- 置信度
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### 4.2.3 置信度计算规则

| 任务次数 | 置信度 | 说明 |
|----------|--------|------|
| 1次 | 0.5 | 初始置信度 |
| 2-3次 | 0.6 | 低置信度 |
| 4-5次 | 0.7 | 中置信度 |
| 6-10次 | 0.8 | 较高置信度 |
| 10次以上 | 0.9 | 高置信度 |

提取咨询者信息
    ↓
提取答复人信息
    ↓

    ↓
更新学习记录
    ↓
完成
```

**自动学习的字段：**
- 咨询者姓名、部门、公司、联系方式
- 答复人姓名、部门

- 所属行业

**执行流程：**
```
用户点击批量学习
    ↓
获取所有任务数据
    ↓
逐条学习任务信息
    ↓
更新学习记录
    ↓
显示学习统计
    ↓
