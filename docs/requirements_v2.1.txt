## 一、项目概述

### 1.1 项目名称
市场咨询任务跟踪工具

### 1.2 项目目标
开发一款面向市场咨询场景的任务跟踪管理工具，实现任务信息解析、记录、查询、跟踪、提醒的一体化管理，提升市场咨询工作的效率和质量。

### 1.3 运行环境
- **操作系统**：Windows
- **技术栈**：Python + PyQt5
- **数据库**：SQLite（本地存储）
- **打包形式**：可执行程序（.exe）

### 1.4 技术集成
| 集成项 | 实现方式 | 说明 |
|--------|----------|------|
| OCR识别 | 本地Tesseract OCR（优先）+ 预留云端API接口 | |
| Outlook邮件 | 直接连接Outlook读取 + .msg文件导入 | |
| MSG邮件文件 | extract-msg库解析 + 无第三方库备用方案 | |
| 企业微信 | 自建应用发消息 + webhook群机器人 | |
| 数据导出 | Excel表格 | |
| **学习积累** | **基于任务数据的智能学习与知识积累** | |
| **重名区分** | **工号字段支持，姓名+工号唯一识别** | ⭐V2.1新增 |

## 二、功能需求

### 2.1 内容导入功能

#### 2.1.1 功能说明
支持多种渠道的内容导入，自动解析为任务信息。

#### 2.1.2 导入方式
| 方式 | 说明 | 优先级 |
|------|------|--------|
| 文本粘贴 | 直接粘贴文字内容 | P0 |
| 图片导入 | OCR识别（本地Tesseract优先） | P0 |
| Outlook邮件 | 直接连接Outlook指定邮件读取 | P0 |
| MSG邮件文件 | 读取导出的.msg文件，解析邮件内容 | P0 |
| 企业微信 | 聊天记录文件导入 + 截图OCR识别 | P1 |

#### 2.1.3 MSG邮件文件导入功能

##### 2.1.3.1 功能概述
支持导入Outlook导出的MSG格式邮件文件，自动解析邮件内容并提取任务相关信息。

##### 2.1.3.2 技术实现
| 组件 | 实现方式 | 说明 |
|------|----------|------|
| 首选方案 | extract-msg库 | 功能完整，解析准确 |
| 备用方案 | 自定义解析器 | 无第三方库时使用 |
| 文件大小限制 | 100MB | 防止大文件导致内存溢出 |

| 咨询者姓名 | 发件人姓名 | sender_name |
| 咨询者联系方式 | 发件人邮箱 | sender_email |

## 五、重名区分与数据刷新优化（V2.1新增）

### 5.1 功能概述

解决实际使用中的两个关键问题：
1. **通讯录重名问题**：姓名相同时，需要工号区分不同联系人
2. **推荐库多模块问题**：同一人擅长多个模块时，需要正确汇总和匹配

### 5.2 通讯录重名区分

#### 5.2.1 问题场景

```
场景1：张三（工号E001）和张三（工号E002）是两个不同的人
场景2：赵六有MAC认证专长，赵六也有802.1x认证专长
```

#### 5.2.2 解决方案

| 字段 | 说明 | 唯一约束 |
|------|------|----------|
| name | 姓名 | - |
| employee_id | 工号（可选） | - |
| **复合唯一** | name + employee_id | ✅ 姓名+工号唯一 |

#### 5.2.3 重名处理流程

```
学习新联系人
    ↓
检查是否存在同名联系人
    ↓
    ├── 无同名 → 直接创建
    │
    └── 有同名
            ├── 工号相同 → 更新记录
            ├── 工号不同 → 创建新记录（区分重名）
            └── 无工号 → 创建新记录（无工号区分）
```

### 5.3 数据刷新逻辑（V2.1优化）

#### 5.3.1 刷新规则

| 场景 | 处理方式 | 说明 |
|------|----------|------|
| 新数据有值，旧数据为空 | 覆盖 | 使用新数据 |
| 新数据有值，旧数据有值 | 覆盖 | **新数据优先** ⭐V2.1 |
| 新数据为空，旧数据有值 | 保留 | 保留旧数据 |

#### 5.3.2 刷新示例

```
旧记录：张三，部门=空，公司=H3C，电话=空，邮箱=空
新数据：张三，部门=技术部，公司=空，电话=13800001111，邮箱=zhangsan@h3c.com

刷新后：张三，部门=技术部，公司=H3C，电话=13800001111，邮箱=zhangsan@h3c.com
```

### 5.4 推荐库多模块映射（V2.1新增）

#### 5.4.1 问题场景

```
任务1：赵六 - MAC认证
任务2：赵六 - 802.1x认证

期望结果：赵六 → MAC认证、802.1x认证（汇总视图）
智能匹配：搜索"MAC认证" → 能关联到赵六
          搜索"802.1x认证" → 也能关联到赵六
```

#### 5.4.2 解决方案

| 字段 | 说明 | 示例 |
|------|------|------|
| key_module | 当前关键模块 | MAC认证 |
| all_modules | 该人所有模块汇总 | MAC认证、802.1x认证 |
| related_modules | 当前任务的相关模块 | MAC认证、其他模块 |

#### 5.4.3 智能匹配逻辑

```python
# 三层匹配
conditions = [
    "key_module LIKE ?",       # 1. 精确匹配当前模块
    "related_modules LIKE ?",  # 2. 在相关模块中匹配
    "all_modules LIKE ?"       # 3. 在汇总模块中匹配 ⭐V2.1
]
```

#### 5.4.4 汇总视图

```python
def get_aggregated_recommendations(key_module):
    """
```
