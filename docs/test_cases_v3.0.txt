### 2.1 通讯录学习测试（10个用例）

#### TC-LEARN-001: 通讯录学习表创建测试
- **测试目的**：验证通讯录学习相关表创建成功
- **前置条件**：数据库连接正常
- **测试步骤**：
  1. 初始化 ContactLearningService
  2. 检查数据库表是否存在
- **预期结果**：contact_learning_records 和 learned_contacts 表创建成功

#### TC-LEARN-002: 从任务学习咨询者信息测试
- **测试目的**：验证从任务数据中学习咨询者信息
- **前置条件**：有待学习的任务数据
- **测试步骤**：
  1. 准备任务数据（包含咨询者信息）
  2. 调用 learn_from_task 方法
  3. 查询 learning_records 表
- **预期结果**：咨询者信息正确写入 learning_records 表

#### TC-LEARN-003: 从任务学习答复人信息测试
- **测试目的**：验证从任务数据中学习答复人信息
- **前置条件**：有待学习的任务数据
- **测试步骤**：
  1. 准备任务数据（包含答复人信息）
  2. 调用 learn_from_task 方法
  3. 查询 learned_contacts 表
- **预期结果**：答复人信息正确写入 learned_contacts 表

#### TC-LEARN-004: 联系人置信度计算测试
- **测试目的**：验证联系人置信度正确计算
- **前置条件**：通讯录学习服务正常
- **测试步骤**：
  1. 学习同一联系人多次（2次、4次、6次、10次）
  2. 检查置信度变化
- **预期结果**：置信度随任务次数递增

| 任务次数 | 置信度 |
|----------|--------|
| 1次 | 0.5 |
| 2-3次 | 0.6 |
| 4-5次 | 0.7 |
| 6-10次 | 0.8 |
| 10次以上 | 0.9 |

#### TC-LEARN-005: 通讯录学习搜索功能测试
- **测试目的**：验证通讯录学习搜索功能
- **前置条件**：有学习记录数据
- **测试步骤**：
  1. 调用 search_contacts 方法
  2. 传入搜索关键词
  3. 检查返回结果
- **预期结果**：返回包含关键词的联系人记录

#### TC-LEARN-006: 按联系人类型筛选测试
- **测试目的**：验证按联系人类型筛选功能
- **前置条件**：有咨询者和答复人记录
- **测试步骤**：
  1. 调用 search_contacts 方法
  2. 设置 contact_type='咨询者'
  3. 检查返回结果
- **预期结果**：只返回咨询者类型记录

#### TC-LEARN-007: 按置信度筛选测试
- **测试目的**：验证按置信度筛选功能
- **前置条件**：有不同置信度的记录
- **测试步骤**：
  1. 调用 search_contacts 方法
  2. 设置 min_confidence=0.7
  3. 检查返回结果
- **预期结果**：只返回置信度>=0.7的记录

#### TC-LEARN-008: 导出到通讯录功能测试
- **测试目的**：验证导出到通讯录功能
- **前置条件**：有学习记录数据
- **测试步骤**：
  1. 选择要导出的联系人
  2. 调用 export_to_contacts 方法
  3. 检查通讯录表数据
- **预期结果**：联系人成功导出到通讯录

#### TC-LEARN-009: 通讯录学习统计功能测试
- **测试目的**：验证通讯录学习统计功能
- **前置条件**：有学习记录数据
- **测试步骤**：
  1. 调用 get_statistics 方法
  2. 检查返回的统计数据
- **预期结果**：返回包含记录数、咨询者数、答复者数、平均置信度等

#### TC-LEARN-010: 重复学习不重复记录测试
- **测试目的**：验证重复学习不会创建重复记录
- **前置条件**：有待学习的任务
- **测试步骤**：
  1. 学习同一任务两次
  2. 检查数据库记录数
- **预期结果**：task_count 递增，但记录数不增加
